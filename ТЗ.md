# Техническое задание: Интерактивное дерево сотрудников

## 1. Общее описание проекта

**Название:** OrgTree — интерактивная организационная структура
**Цель:** Веб-приложение для визуализации и управления организационной структурой коммерческого блока (Узум Авто). Удобный просмотр для HR и сотрудников, редактирование — только для авторизованных пользователей.

---

## 2. Технологический стек

| Слой | Технология |
|------|-----------|
| Frontend | React + TypeScript + Vite |
| UI-библиотека | Tailwind CSS + shadcn/ui |
| Визуализация дерева | React Flow (или D3.js) |
| Backend | Node.js + Express |
| База данных | SQLite (для простоты деплоя, легко мигрировать на PostgreSQL) |
| ORM | Prisma |
| Аутентификация | JWT (JSON Web Tokens) |
| Деплой | Docker (опционально) |

> **Почему этот стек:** Проект запускается в VS Code, минимум зависимостей, SQLite не требует отдельного сервера, всё заводится одной командой `npm run dev`.

## 4. Модель данных (Prisma Schema)

```prisma
model User {
  id        Int      @id @default(autoincrement())
  email     String   @unique
  password  String                    // bcrypt hash
  name      String
  role      Role     @default(VIEWER)
  createdAt DateTime @default(now())
}

enum Role {
  ADMIN       // Полный доступ (Дильшод / назначенные)
  EDITOR      // Может редактировать обязанности, перемещать сотрудников
  VIEWER      // Только просмотр (HR, коллеги)
}

model Employee {
  id           Int        @id @default(autoincrement())
  fullName     String                  // ФИО
  position     String                  // Должность
  department   String                  // Блок (Коммерческий отдел, Модерация и Суппорт, Финансовый Отдел)
  duties       String?                 // Функционал (текст)
  managerId    Int?                    // Руководитель (ссылка на Employee)
  manager      Employee?  @relation("Subordinates", fields: [managerId], references: [id])
  subordinates Employee[] @relation("Subordinates")
  photoUrl     String?                 // Фото (опционально)
  createdAt    DateTime   @default(now())
  updatedAt    DateTime   @updatedAt
}

model AuditLog {
  id         Int      @id @default(autoincrement())
  userId     Int
  action     String                   // "MOVE", "EDIT_DUTIES", "ADD", "DELETE"
  target     String                   // Кто был изменён
  details    String?                  // Что именно изменилось
  createdAt  DateTime @default(now())
}
```

## 5. Функциональные требования

### 5.1 Визуализация дерева

- **Интерактивное дерево** с узлами-карточками сотрудников
- Каждая карточка показывает: **ФИО**, **Должность**, **Блок** (цветовая метка)
- Дерево раскладывается сверху вниз (корень — Егор Абрамец / Мамаджанов Дильшод)
- **Zoom** (колёсико мыши) и **Pan** (перетаскивание холста)
- **Сворачивание/разворачивание** веток по клику на узел
- Цветовое кодирование блоков:
  - Коммерческий отдел — синий
  - Модерация и Суппорт — зелёный
  - Финансовый Отдел — оранжевый
- **Адаптивность**: на мобильных — вертикальный список с вложенностью

### 5.2 Карточка сотрудника (модальное окно)

По клику на узел открывается модалка:
- ФИО, должность, блок
- Руководитель (ссылка-переход к его узлу)
- Список обязанностей (каждая — отдельный пункт)
- Подчинённые (список со ссылками)
- Фото (если загружено)
- Кнопки «Редактировать» / «Переместить» (видны только ADMIN и EDITOR)

### 5.3 Поиск и фильтрация

- **Поиск** по ФИО (с подсветкой найденного узла на дереве)
- **Фильтр** по блоку/отделу
- **Фильтр** по руководителю

### 5.4 Аутентификация и роли

| Роль | Просмотр | Редактирование обязанностей | Перемещение сотрудников | Управление доступами | Добавление/удаление сотрудников |
|------|----------|---------------------------|------------------------|---------------------|-------------------------------|
| VIEWER | + | - | - | - | - |
| EDITOR | + | + | + | - | - |
| ADMIN | + | + | + | + | + |

- Логин по email + пароль
- JWT-токен хранится в httpOnly cookie
- При первом запуске автоматически создаётся аккаунт администратора (из seed)

### 5.5 Управление доступами (Админ-панель)

- Список пользователей с их ролями
- Кнопки: изменить роль, деактивировать пользователя
- Создание нового пользователя (email, пароль, роль)
- История изменений (Audit Log) — кто, когда, что изменил

### 5.6 Редактирование обязанностей

- Inline-редактирование текста обязанностей в модалке сотрудника
- Каждая обязанность — отдельная строка, можно добавлять/удалять
- Сохранение через API с записью в Audit Log

### 5.7 Перемещение сотрудников

- **Drag & Drop** на дереве: перетащить узел сотрудника к другому руководителю
- Или через модалку: выбрать нового руководителя из выпадающего списка
- Подчинённые перемещаемого сотрудника остаются за ним (перемещается вся ветка)
- Подтверждение действия перед сохранением
- Запись в Audit Log

## 6. API Endpoints

### Auth
```
POST   /api/auth/login          — Логин, возвращает JWT
POST   /api/auth/logout         — Logout (удаление cookie)
GET    /api/auth/me             — Текущий пользователь
```

### Employees
```
GET    /api/employees           — Список всех (дерево)
GET    /api/employees/:id       — Один сотрудник с подчинёнными
POST   /api/employees           — Добавить сотрудника (ADMIN)
PUT    /api/employees/:id       — Обновить данные (ADMIN, EDITOR)
PATCH  /api/employees/:id/move  — Переместить к другому руководителю (ADMIN, EDITOR)
DELETE /api/employees/:id       — Удалить сотрудника (ADMIN)
```

### Access Management
```
GET    /api/users               — Список пользователей (ADMIN)
POST   /api/users               — Создать пользователя (ADMIN)
PATCH  /api/users/:id/role      — Изменить роль (ADMIN)
DELETE /api/users/:id           — Деактивировать (ADMIN)
```

### Audit
```
GET    /api/audit               — Лог изменений (ADMIN)
```

## 8. UI/UX требования

### Дизайн
- Светлая тема, минималистичный стиль
- Карточки сотрудников: белые со скруглёнными углами, лёгкая тень
- Цветная полоска сверху карточки = цвет блока
- При наведении — подсветка карточки и связей
- Шрифт: Inter или системный

### Навигация
- Верхняя панель: логотип, поиск, кнопка входа / имя пользователя
- Левая панель (складная): фильтры, список сотрудников текстом
- Центр: интерактивное дерево
- Кнопки zoom +/- и «вписать в экран» в углу

## 9. Запуск проекта

```bash
# 1. Установка зависимостей
npm install

# 2. Инициализация БД и загрузка данных из Excel
npm run db:setup

# 3. Запуск в режиме разработки (frontend + backend)
npm run dev
```

Один `npm run dev` поднимает:
- Frontend на `http://localhost:5173`
- Backend на `http://localhost:3001`
